# Research-driven architecture
From 01-RESEARCH.md standard stack:
- WordPress 6.9.4-php8.2-fpm-alpine image (official)
- Nginx Alpine reverse proxy (performance standard)
- MySQL 8.0 official image (production-compatible)
- WP-CLI separate container with shared volumes
- Volume mounting for hot-reload: ./wp-content → /var/www/html/wp-content
</context>

<task type="auto">
  <name>Task 1: Create docker-compose.yml with multi-container setup</name>
  <files>docker-compose.yml</files>
  <action>
Create Docker Compose file with 4 services:

1. **wordpress** service:
   - Image: wordpress:6.9.4-php8.2-fpm-alpine
   - Container: artricenter_wp
   - Environment: WORDPRESS_DB_HOST=db:3306, WORDPRESS_DB_NAME=artricenter, WORDPRESS_DB_USER=wpuser, WORDPRESS_DB_PASSWORD=wpsecret
   - Volumes: ./wp-content:/var/www/html/wp-content (hot-reload), ./docker/php/uploads.ini:/usr/local/etc/php/conf.d/uploads.ini
   - Depends on: db
   - Network: artricenter_network

2. **nginx** service:
   - Image: nginx:alpine
   - Container: artricenter_nginx
   - Ports: 8080:80 (avoid conflicts)
   - Volumes: ./wp-content:/var/www/html/wp-content, ./docker/nginx/default.conf:/etc/nginx/conf.d/default.conf
   - Depends on: wordpress
   - Network: artricenter_network

3. **db** service:
   - Image: mysql:8.0
   - Container: artricenter_db
   - Environment: MYSQL_DATABASE=artricenter, MYSQL_USER=wpuser, MYSQL_PASSWORD=wpsecret, MYSQL_ROOT_PASSWORD=rootsecret
   - Volumes: db_data:/var/lib/mysql (persistent)
   - Network: artricenter_network

4. **wpcli** service:
   - Image: wordpress:cli-php8.2
   - Container: artricenter_wpcli
   - Volumes: ./wp-content:/var/www/html/wp-content
   - Depends on: db, wordpress
   - Network: artricenter_network
   - No command (interactive use)

5. **Volumes and networks**:
   - Named volume: db_data for MySQL persistence
   - Network: artricenter_network (bridge driver)

Use version: '3.9' syntax. Include restart: unless-stopped for all services.
  </action>
  <verify>
    <automated>docker compose config > /dev/null 2>&1 && echo "Valid Docker Compose configuration"</automated>
  </verify>
  <done>

  </done>

Configuration must handle WordPress permalink structure and PHP execution correctly.
